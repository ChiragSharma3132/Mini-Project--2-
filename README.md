# Mini-Project--2-
This project implements a simplified relational database engine designed to demonstrate the internal working of database management systems. The goal of this project is to help understand how core database components such as query parsing, indexing, storage management, and query execution function internally.

Unlike traditional databases that operate as black-box systems, this project focuses on building the core architecture of a database engine from scratch. It provides a hands-on understanding of how queries are processed, how data is stored, and how indexing improves retrieval efficiency.

The system supports basic SQL operations and simulates the internal workflow of a relational database system including query parsing, execution planning, and result retrieval.

# 🎯 Objectives

Understand the internal architecture of relational database systems

Implement a query parser, planner, and execution engine

Learn concepts of data storage and indexing

Gain practical experience in system-level programming

Demonstrate how database queries are processed internally

# ⚙️ Features

Support for basic SQL operations:

CREATE

INSERT

SELECT

File-based data storage

Basic indexing mechanism (B+ Tree)

Query parsing and execution pipeline

Simple command-line interface (CLI) for database interaction
